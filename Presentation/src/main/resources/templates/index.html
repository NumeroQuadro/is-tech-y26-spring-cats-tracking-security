Lab Task: Adding Spring to the Service

This lab task requires integrating Spring into the service created in a previous lab. The service must provide an HTTP interface (REST API) for accessing information about specific cats and their owners, as well as for retrieving filtered information (e.g., finding all ginger cats).

Requirements

  • The service must not expose JPA entities through the HTTP interface. Instead, use separate wrapper classes (DTOs).
  • Services and DAOs should become Spring Beans using Dependency Injection (@Autowired).
  • DAOs should extend JpaRepository and utilize templated Spring Data JPA methods.

Demonstration of the endpoints' functionality via HTTP requests is required upon submission (using tools such as Postman is recommended).

Key Considerations

It's important to adhere to best practices in software development, including separation of concerns by using DTOs for HTTP communications and leveraging Spring's Dependency Injection for managing service and DAO beans. This approach enhances the maintainability and scalability of the service.
